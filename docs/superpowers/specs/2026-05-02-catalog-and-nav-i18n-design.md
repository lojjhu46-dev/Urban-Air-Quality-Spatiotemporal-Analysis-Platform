# 内置目录与侧边栏导航国际化设计

**目标**

在切换到 `zh-CN` 时，让历史数据 Agent 中内置目录的大洲、国家/地区、省/州/行政区和城市全部显示中文；在 `en` 时保持当前英文显示。同时让左侧页面导航标签跟随语言切换。

**范围**

- 仅覆盖当前代码库里的内置目录。
- 不改变英文目录值在状态、匹配和 API 查询中的作用。
- 不尝试翻译 API 返回的动态地名。

**方案**

1. 为内置目录增加语言感知的显示层：
   - 大洲、国家/地区、省/州/行政区、城市都通过 helper 读取显示名。
   - `AgentCityOption` 增加完整的 `display_*` 方法，并让 `path_label_for_language()` 走统一显示逻辑。
   - Streamlit 下拉框继续使用英文稳定值作为 `options`，只通过 `format_func` 渲染当前语言文本。

2. 为侧边栏页面导航增加可翻译的自定义渲染：
   - 在一个统一 helper 中声明页面路径与翻译 key。
   - 各页在 sidebar 调用同一个渲染函数输出 `st.page_link(...)`。
   - 通过 Streamlit 配置隐藏默认 sidebar 页面导航，避免出现两套导航。

**验证**

- 新增测试覆盖中英两种语言下的目录显示名。
- 新增测试覆盖导航项标签翻译。
- 回归运行 `tests/test_agent_interaction.py`、`tests/test_i18n.py`，并补充导航 helper 的测试模块。
